00 ｜ Build–Measure–Learn ループの設計思想
🔨
Build
小さく作る
完璧を目指さない。アプリ開発より先に「LINEとGoogleで動く最小の仕組み」を今週中に起動する。
📊
Measure
正直に測る
7日継続率・動画視聴率・フィードバック数。感覚ではなく数字で判断する。毎週月曜にKPIを記録。
💡
Learn
素早く改善する
データから「なぜ続いた／続かなかった」を分析。次のBuildに反映。このループを月1回転させる。
🎯 MVPの判断基準
7日継続率が70%以上 → 次のループへ進む。70%未満 → リマインダーのタイミングや動画の長さを改善して再測定。30日継続率が50%以上 → 有料化フェーズへ移行する判断ができる。

03 ｜ Cursor × GitHub 自動化ワークフロー
1
リポジトリをCloneしてCursorで開く
まずGitHubでリポジトリを作成。Cursorで「Open Folder」でそのリポジトリを開く。これ以降、コード編集はすべてCursorの中で完結する。
git clone https://github.com/asa-ai-lab/breathing-habit.git cd breathing-habit
GitHub Cursor
2
CursorのAIチャットでGASコードを生成
Cursorのチャット欄に「やりたいこと」を日本語で書くだけでコードを生成してくれる。エラーもAIが直してくれるのでコーディング経験不要。
// Cursorチャットへの指示例： "毎朝7時にLINE公式アカウントから 登録ユーザー全員にメッセージを送る GASスクリプトを書いてください"
Cursor AI GAS
3
動作確認してGitHubにcommit & push
コードが動いたらCursorのSource Controlパネル（GitHubマーク）から変更をcommitしてpushする。差分が自動で記録されるので「どの時点に戻るか」がいつでも選べる。
git add gas/reminder_line.gs git commit -m "feat: LINEリマインダー送信スクリプト追加" git push origin main
commit Source Control
4
Claudeでプロンプトを磨き、promptsフォルダに保存
「先生からの一言」のプロンプトテンプレートをClaudeで作成・改善。良いバージョンをprompts/フォルダにmarkdownで保存してcommitする。プロンプトもバージョン管理できる。
# prompts/daily_message_template.md に保存 git add prompts/daily_message_template.md git commit -m "prompt: 月曜テーマ（心理学）の一言テンプレート更新"
Claude commit
5
KPIデータを毎週追記 → Claudeで分析
毎週月曜にkpi_weekly_log.csvにデータを追記してcommit。蓄積されたデータをClaudeに渡して「継続率が下がった週の原因分析」を依頼する。データがGitHubにあるので会話にそのまま貼れる。
git add data/kpi_weekly_log.csv git commit -m "data: Week3 KPI記録 継続率68%"
Claude分析 データ管理
04 ｜ 「先生からの一言」Claudeプロンプトテンプレート
prompts/daily_message_template.md ｜ 週テーマ別テンプレート
あなたはヨガインストラクター「はっち」として、 生徒へ毎日の呼吸習慣をサポートする短いメッセージを書いてください。 【今日のテーマ】{{theme}} 【週テーマ例】 - 月：心理（自己効力感・習慣化の仕組み） - 火：生理（体への具体的な変化・迷走神経・HRV） - 水：古典知恵（プラーナヤーマ・アーユルヴェーダ） - 木：脳科学（コルチゾール・デフォルトモードネットワーク） - 金：実践tips（if-thenプランニング・タイミング設計） - 土：振り返り（今週の気づきを一言で） - 日：意図設定（来週への小さな宣言） 【生徒の状況】{{user_context}} （例：継続率70%、7日目、やり方が不安という声が多い） 【制約】 - 150文字以内 - 押しつけがましくなく、温かく寄り添う文体 - 科学的な根拠を1文だけ添える - 最後に今日の呼吸法への具体的な誘い文句を入れる - 絵文字を1〜2個使う 【とっとさんの文体参考】 「〜だよ」「〜してみてね」「〜だから大丈夫」という 沖縄の空気感と優しさが滲む語り口
💡 プロンプトをGitHubで管理する利点
「2週間前のバージョンの方がよかった」という時に即座に戻せる。また蓄積されたプロンプトのバージョン履歴は、将来他のインストラクターにライセンスする際の「知的財産」になる。
